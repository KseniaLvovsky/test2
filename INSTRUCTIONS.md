# Instructions

Welcome to the practice assessment.

This repository simulates a startup-style QA assignment with some ambiguity, a manual QA fundamentals section, an API testing section, and one small optional implementation task.

## What this practice is evaluating

- QA fundamentals
- API testing concepts
- test case design
- bug reporting
- handling ambiguity
- structured written communication
- basic problem solving

## Rules for this practice simulation

Pretend the following rules were given by the company:

- Read all instructions before starting.
- For Task 1 and Task 2, do not use the internet.
- Do not copy code from external sources.
- Write answers clearly and in your own words.
- Keep answers concise but complete.
- Optional task does not require any external libraries.

## Your goal

Complete the files in `answers/` as if you were submitting this assignment.

## What is allowed in this practice repo

Because this is your training copy, you may:
- inspect the file structure
- practice editing directly in GitHub
- clone locally and work in your IDE
- compare your work to the reference solution afterward

## Suggested submission format

In a real assessment, companies often want:
- your completed answers committed to the repository
- a clean commit history
- readable markdown
- clear reasoning rather than overly long explanations
