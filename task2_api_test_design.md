# Task 2 — API Testing Design

Use the materials in `resources/api_spec.md` and `resources/sample_requests.md`.

Document your answers in `answers/task2_answers.md`.

## Scenario

You are testing an API endpoint that creates an audio conversion job for uploaded text.

### Endpoint
`POST /v1/audio/jobs`

### Purpose
Creates a job that converts submitted text into audio.

## Your tasks

1. Write at least 12 test cases for this endpoint.
2. Include positive, negative, edge, and validation scenarios.
3. List the most important status codes you would expect.
4. Identify at least 5 risks or ambiguities in the API spec.
5. Describe what data you would prepare before testing.
6. Write 3 strong bug reports based on the intentionally flawed API responses in `resources/sample_requests.md`.

## Notes

You do not need to automate this task.
Focus on test thinking, coverage, clarity, and realistic QA reasoning.
